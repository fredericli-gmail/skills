# Skill: 簡報（簡報撰寫與品質檢查專家）

> 🎯 **核心使命**：產出「結構統一、版面整齊、文字不溢出、元素不重疊」的高品質簡報。
> 每頁調整後必須通過「逐頁截圖驗證 → 文字不溢出 → 文字不重疊」三大規則。
>
> 🛠 **技術實作**：統一使用 **`python-pptx`**（搭配 LibreOffice + pdf2image 做截圖驗證）。
> 所有常數、輔助函式、驗證 / 修復程式碼請參考 [PPTX-RECIPES.md](PPTX-RECIPES.md)。

## Usage Trigger

- 使用者要求撰寫、產生或美化簡報（PowerPoint / Keynote / pptx）
- 使用者說「/簡報」、「/check-deck」、「簡報撰寫」、「投影片美化」、「簡報檢查」
- 已完成簡報需要全面健檢、批次修復
- 需要依照統一視覺規範產出新簡報頁面

---

## 1. 簡報品質規範（最高原則）

### 1.1 三大品質規則（必要，不可妥協）

> ⚠️ **任何一頁不滿足任一規則，必須修正到滿足為止，不可放行。**

#### 規則 1：逐頁截圖驗證
- 每次調整完一頁後，**必須**執行 `verify_slide_visual` 產出該頁截圖
- 對照本規範逐項檢查，若有任一項不滿足立即調整
- 每次修復後必須重新截圖驗證，直到通過

#### 規則 2：文字不溢出（No Overflow）
- 任何文字框內的文字不可超出框線
- 修復順序（擇一或組合）：
  1. **縮小字體**（優先，最不破壞版面）
  2. **縮短文字**（其次，需維持語意）
  3. **擴大容器**（最後，需確保不與其他元素重疊）

#### 規則 3：文字不重疊（No Overlap）
- 任何兩段文字、元素之間不可重疊
- 修復順序（擇一或組合）：
  1. **調整位置**（優先）
  2. **縮小字體**
  3. **縮短文字**

## 2. 標題區間距（強制）

| 項目 | 規範 |
|------|------|
| 頂部色帶 | 全寬矩形，`height < 15pt`、`top < 2` |
| 圓形圖標（Oval） | `~40×40pt`、`left < 50` |
| **色帶 → 圓形圖標 間距** | **≥ 20pt**（不足必須修正） |

**修正方式**：若間距不足 20pt，將「圓形圖標、頁碼文字、標題、副標、分隔線」**整組統一下移**至滿足 20pt（目前版本已下移 8pt，間距由 13pt → 21pt）。

## 3. 版面結構規範

### 3.1 投影片尺寸
- **960 × 540pt**（寬螢幕 16:9）

### 3.2 標題區佈局

| 元素 | 位置 / 尺寸 | 樣式 |
|------|-------------|------|
| 頂部色帶 | 全寬 × ~9pt 高 | 填色 `#EA580C`（主色） |
| 圓形圖標（Oval） | `left=36, top≈30, 40×40pt` | 填色 `#EA580C`，白色頁碼數字 |
| 標題文字 | `left=86, top≈33` | 28pt 粗體、`#0F1B3D` |
| 副標文字 | `left=86, top≈64` | 14pt、`#64748B` |
| 分隔線 | `left=36, top≈98, 寬 888pt, 高 0.5pt` | `#E2E8F0` |

### 3.3 內容區域
- **起始 y ≈ 111**（分隔線下方）
- **左右邊距** ≈ 36pt

### 3.4 頁尾

| 元素 | 位置 | 樣式 |
|------|------|------|
| 左下機密標註 | `left=36, top≈515` | 「營運秘密 嚴勿外流（安全等級 - 機密）」、10pt、`#94A3B8` |
| 右下頁碼 | `left≈850, top≈515` | 格式 `XX / 28`、10pt、`#94A3B8` |
| 底部裝飾線 | `left=36, top=510, 寬 888pt` | `#E2E8F0` |

## 6. 卡片風格

- 白色或淡色背景矩形

- 卡片內：粗體標題行 + 一般描述文字
- 卡片間距**均勻排列**（2 欄或 3 欄佈局）

## 8. 全簡報健檢工作流（/check-deck）

### 8.1 流程步驟

```
① 掃描
   ↓ verify_slides 分批掃描所有頁面（收集所有元素位置與尺寸）
② 標題間距檢查
   ↓ 遍歷所有頁面確認「色帶 → 圓形圖標」間距 ≥ 20pt
③ 彙報問題
   ↓ 列出：間距不足 / 文字溢出 / 文字重疊
④ 等待使用者確認（OKOKYES）
⑤ 自動修復
   ↓ 批量修正所有問題
⑥ 逐頁驗證
   ↓ verify_slide_visual 截圖逐頁確認
⑦ 最終報告
   ↓ 彙報修復結果
```

### 8.2 彙報格式

```
【簡報健檢報告】

════════════════════════════════════════════════════════════════

📊 掃描摘要
├── 總頁數：{n}
├── 🔴 標題間距不足：{n} 頁
├── 🔴 文字溢出：{n} 處
├── 🔴 文字重疊：{n} 處
└── ✅ 正常頁面：{n} 頁

════════════════════════════════════════════════════════════════

🔴 問題清單

┌────────┬──────────────┬──────────────────────────────────────┐
│ 頁碼   │ 類型         │ 詳細說明                             │
├────────┼──────────────┼──────────────────────────────────────┤
│ P03    │ 標題間距     │ 色帶 → 圓形圖標 間距 13pt（< 20pt）  │
│ P07    │ 文字溢出     │ 卡片 3 標題超出右邊界 12pt           │
│ P12    │ 文字重疊     │ 副標與內容第一段重疊 5pt             │
└────────┴──────────────┴──────────────────────────────────────┘

════════════════════════════════════════════════════════════════

將執行以下修復：
1. P03 → 整組下移 8pt
2. P07 → 縮小標題字級 1pt 或縮短為「...」
3. P12 → 內容起始 y 下移至 111pt

請輸入「OKOKYES」啟動批量修復，或輸入其他內容繼續對話。
```

### 8.3 修復完成報告

```
【簡報健檢 - 修復完成】

✅ 所有問題已修復
✅ 所有頁面已通過 verify_slide_visual 截圖驗證

📊 修復摘要
├── 標題間距修正：{n} 頁
├── 文字溢出修正：{n} 處
└── 文字重疊修正：{n} 處

共完成 {n} 項修復，全簡報符合品質規範。
```

## 9. 撰寫新簡報的標準流程

1. **建立簡報骨架**：尺寸 960×540pt、套用統一母片（頂部色帶 + 標題區 + 分隔線 + 頁尾）
2. **逐頁產出內容**：依本規範放置標題、副標、卡片、頁碼
3. **每頁完成立即驗證**：`verify_slide_visual` 截圖 → 檢查三大規則 → 不通過立即修復
4. **全簡報完成後**：執行 `/check-deck` 做一次全面健檢
5. **最終交付前**：確保沒有任一頁存在「間距不足 / 文字溢出 / 文字重疊」

---

## 10. 禁止行為

- ❌ 禁止未截圖驗證就宣告某頁完成
- ❌ 禁止放行任一項「文字溢出」或「文字重疊」
- ❌ 禁止標題區間距 < 20pt 的頁面
- ❌ 禁止混用非 Microsoft JhengHei 的字型
- ❌ 禁止使用規範外的色碼（需新增色彩時先與使用者確認）
- ❌ 禁止未取得 `OKOKYES` 就執行批量修復
- ❌ 禁止接受 `OKOKYES` 以外的確認詞彙

## 11. 與其他 Skill 的關係

- 本 Skill 為**獨立工作流**，不自動串接其他 Skill
