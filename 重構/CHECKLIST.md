## 1. 重構前檢查

### 1.1 環境準備

| 項目 | 檢查內容 | 指令 |
|------|---------|------|
| [ ] Git 狀態乾淨 | 無未提交的變更 | `git status` |
| [ ] 在正確分支 | 不在 main/master 直接重構 | `git branch` |
| [ ] 最新程式碼 | 已拉取最新版本 | `git pull` |

### 1.2 測試準備

| 項目 | 檢查內容 | 指令 |
|------|---------|------|
| [ ] 測試可執行 | 專案測試可正常執行 | `./mvnw test` |
| [ ] 測試通過 | 重構前所有測試通過 | 檢查測試結果 |
| [ ] 測試覆蓋 | 待重構程式碼有測試覆蓋 | 檢查測試類別 |

### 1.3 範圍確認

| 項目 | 檢查內容 |
|------|---------|
| [ ] 明確範圍 | 已確定要重構的檔案/方法 |
| [ ] 影響評估 | 已評估可能受影響的程式碼 |
| [ ] 計畫確認 | 使用者已確認重構計畫（OKOKYES） |

## 2. 重構中檢查

### 2.1 每個重構步驟

```
執行重構
    ↓
[ ] 編譯通過？ ──否──→ 修正編譯錯誤
    ↓ 是
[ ] 測試通過？ ──否──→ 檢查是否改變行為
    ↓ 是
[ ] 立即 Commit
    ↓
下一個重構步驟
```

### 2.2 Commit 檢查

| 項目 | 檢查內容 |
|------|---------|
| [ ] 單一職責 | 每個 commit 只做一件事 |
| [ ] 可獨立回滾 | 這個 commit 可以獨立 revert |
| [ ] 訊息清楚 | commit 訊息說明重構內容 |
| [ ] 格式正確 | 使用 `refactor:` 前綴 |

### 2.3 程式碼風格檢查

針對專案規範進行檢查：

| 項目 | 檢查內容 | 嚴重度 |
|------|---------|-------|
| [ ] 無 Lombok | 沒有 @Data, @Getter 等註解 | 🔴 必須 |
| [ ] 無 Lambda | 沒有使用 `->` 運算子 | 🔴 必須 |
| [ ] 無 Stream | 沒有 .stream(), .map() 等 | 🔴 必須 |
| [ ] 有繁中註解 | 每行邏輯程式碼有註解 | 🟡 建議 |
| [ ] 手寫 Getter/Setter | POJO 手動撰寫存取方法 | 🔴 必須 |

## 3. 重構後檢查

### 3.1 功能驗證

| 項目 | 檢查內容 | 指令 |
|------|---------|------|
| [ ] 編譯成功 | 專案可正常編譯 | `./mvnw compile` |
| [ ] 測試通過 | 所有測試皆通過 | `./mvnw test` |
| [ ] 行為不變 | 業務邏輯未被改變 | 人工確認 |

### 3.2 程式碼品質

| 項目 | 檢查內容 | 目標 |
|------|---------|------|
| [ ] 方法長度 | 重構後方法行數 | < 50 行 |
| [ ] 巢狀深度 | if/for 巢狀層數 | ≤ 3 層 |
| [ ] 重複消除 | 重複程式碼已提取 | 0 處重複 |
| [ ] 命名清楚 | 新方法/類別命名合理 | 人工確認 |

### 3.3 文件更新

| 項目 | 檢查內容 |
|------|---------|
| [ ] 註解更新 | 若有變更邏輯，註解已同步更新 |
| [ ] 無過時註解 | 沒有描述舊邏輯的註解 |
| [ ] 方法文件 | 新提取的 public 方法有 Javadoc |

### 3.4 Git 最終確認

| 項目 | 檢查內容 | 指令 |
|------|---------|------|
| [ ] 全部提交 | 無未提交的變更 | `git status` |
| [ ] Commit 歷史 | 每個 commit 都是有意義的重構 | `git log --oneline` |
| [ ] 可回滾 | 確認每個 commit 可獨立 revert | 人工確認 |

## 4. 常見錯誤檢查

### 4.1 重構常見錯誤

| 錯誤 | 說明 | 如何避免 |
|------|------|---------|
| 改變行為 | 重構時不小心修改業務邏輯 | 執行測試驗證 |
| 過度重構 | 一次改太多，難以追蹤問題 | 小步快跑 |
| 忘記提交 | 多個重構混在一個 commit | 每步都 commit |
| 破壞編譯 | 重構後專案無法編譯 | 每步都編譯 |
| 遺漏參照 | 移動方法後有呼叫遺漏 | IDE 重構工具 |

### 4.2 程式碼品質問題

| 問題 | 檢測方式 |
|------|---------|
| 提取的方法太小 | 方法只有 1-2 行且無重複使用 |
| 提取的方法太大 | 方法仍超過 30 行 |
| 命名不當 | 方法名稱無法反映其功能 |
| 過度抽象 | 為了重構而建立不必要的介面/類別 |

### 4.3 規範違規檢查

```bash
# 檢查是否誤用 Lombok
grep -rn "@Data\|@Getter\|@Setter\|@Builder" --include="*.java" src/

# 檢查是否誤用 Lambda
grep -rn "\->" --include="*.java" src/ | grep -v "//\|/\*"

# 檢查是否誤用 Stream
grep -rn "\.stream()\|\.map(\|\.filter(\|\.forEach(" --include="*.java" src/
```

## 5. 快速檢查指令

### 5.1 重構前

```bash
# 確認 Git 狀態
git status

# 確認測試通過
./mvnw test -q

# 統計目標檔案行數
wc -l [目標檔案]
```

### 5.2 重構中

```bash
# 編譯檢查
./mvnw compile -q

# 快速測試（指定類別）
./mvnw test -Dtest=[測試類別] -q

# 提交重構
git add [檔案]
git commit -m "refactor: [說明]"
```

### 5.3 重構後

```bash
# 完整編譯
./mvnw compile

# 完整測試
./mvnw test

# 檢查 commit 歷史
git log --oneline -10

# 檢查未提交變更
git status

# 檢查違規語法
grep -rn "@Data\|@Getter\|\.stream()\|\->" --include="*.java" src/ | grep -v "//\|/\*"
```

## 7. 緊急回滾程序

當重構出現問題需要回滾時：

### 7.1 回滾單一 Commit

```bash
# 檢視要回滾的 commit
git log --oneline -5

# 回滾特定 commit（保留變更在工作區）
git revert [commit-hash] --no-commit

# 或直接回滾並建立新 commit
git revert [commit-hash]
```

### 7.2 回滾多個 Commit

```bash
# 回滾到特定版本（保留變更在工作區）
git reset --soft [commit-hash]

# 或完全回滾（丟棄所有變更）
git reset --hard [commit-hash]
```

### 7.3 回滾檢查清單

```
【緊急回滾檢查】

□ 已確認需要回滾的 commit
□ 已通知相關人員
□ 回滾後編譯通過
□ 回滾後測試通過
□ 已記錄回滾原因
```

## 6. JavaScript 專用檢查

### 6.1 重構前檢查

| 項目 | 檢查內容 | 指令 |
|------|---------|------|
| [ ] 語法正確 | JS 檔案無語法錯誤 | `node --check [檔案]` |
| [ ] 無執行錯誤 | 瀏覽器 Console 無報錯 | 開啟開發者工具 |
| [ ] Git 狀態乾淨 | 無未提交的變更 | `git status` |

### 6.2 重構中檢查

#### 每個重構步驟

```
執行 JS 重構
    ↓
[ ] 語法檢查通過？ ──否──→ 修正語法錯誤
    ↓ 是
[ ] 瀏覽器測試？ ──否──→ 檢查是否改變行為
    ↓ 是
[ ] 立即 Commit
    ↓
下一個重構步驟
```

#### JavaScript 程式碼風格檢查

| 項目 | 檢查內容 | 嚴重度 |
|------|---------|-------|
| [ ] 無箭頭函式 | 沒有使用 `=>` 語法 | 🔴 必須 |
| [ ] 無 let/const | 使用 `var` 宣告變數 | 🟡 建議 |
| [ ] 無模板字串 | 沒有使用 `` ` `` 反引號 | 🟡 建議 |
| [ ] 有繁中註解 | 每行邏輯程式碼有註解 | 🟡 建議 |
| [ ] 無全域污染 | 使用 IIFE 或模組模式封裝 | 🟡 建議 |
| [ ] 無過多 console | 移除不必要的 console.log | 🟢 可選 |

### 6.3 重構後檢查

| 項目 | 檢查內容 | 指令 |
|------|---------|------|
| [ ] 所有 JS 語法正確 | 所有檔案通過語法檢查 | 見下方批次指令 |
| [ ] 瀏覽器功能正常 | 頁面功能正常運作 | 人工測試 |
| [ ] 專案編譯成功 | Maven 編譯無錯誤 | `./mvnw compile` |
| [ ] 行為不變 | 業務邏輯未被改變 | 人工確認 |

### 6.4 JavaScript 快速檢查指令

#### 重構前

```bash
# 確認 Git 狀態
git status

# 檢查目標 JS 檔案語法
node --check [目標檔案]

# 統計檔案行數
wc -l src/main/resources/static/js/*.js | sort -rn
```

#### 重構中

```bash
# 語法檢查（單一檔案）
node --check [檔案路徑]

# 提交重構
git add [檔案]
git commit -m "refactor: [說明]"
```

#### 重構後

```bash
# 批次語法檢查（所有 JS 檔案）
for file in src/main/resources/static/js/*.js; do
  if node --check "$file" 2>&1 | grep -q "SyntaxError"; then
    echo "❌ $file 語法錯誤"
  else
    echo "✅ $file"
  fi
done

# 檢查違規語法
echo "=== 檢查箭頭函式 ==="
grep -rn "=>" --include="*.js" src/main/resources/static/js/ | grep -v "//\|/\*\|.min.js" | head -10

echo "=== 檢查 let/const ==="
grep -rn "\blet\b\|\bconst\b" --include="*.js" src/main/resources/static/js/ | grep -v "//\|/\*\|.min.js" | head -10

echo "=== 檢查模板字串 ==="
grep -rn '`' --include="*.js" src/main/resources/static/js/ | grep -v "//\|/\*\|.min.js" | head -10

# 檢查 commit 歷史
git log --oneline -10

# 檢查未提交變更
git status
```
